# OctoAcme — Retrospective & Continuous Improvement

**Document Owner:** Project Manager  
**Contributors:** All team members (Developers, QA Lead, DevOps Engineer, UX Designer, Technical Writer, PdM)

## Purpose
Capture learnings and convert them into actionable improvements.

## When
After each sprint, release, or important milestone. Also after incidents.

## Participants
All delivery team roles should be invited: PM (facilitator), PdM, Developers, QA Lead, DevOps Engineer, UX Designer, and Technical Writer.

## Structure
- What went well
- What could be improved
- Action items (owner, due date)
- Follow-up on previous action items

## Running a Retrospective
- Timebox: 45–75 minutes depending on team size
- Use an anonymous idea board if needed to encourage candor
- Prioritize 2–3 top action items to avoid overload
- Every action item must have a named owner and due date before the retrospective closes

## Tracking Improvements
- Add action items to the project backlog or issues with clear owners and timelines
- Review outstanding actions in the weekly PM sync
- QA Lead tracks quality-related action items; DevOps Engineer tracks infrastructure action items
- Technical Writer updates process documentation to reflect process improvements agreed in retrospectives

## Quality Gate — Action Item Follow-up

Before a retrospective is considered **closed**, all of the following must be true:

- [ ] All action items from this retrospective have a named owner and due date
- [ ] Action items have been added to the project backlog or issue tracker
- [ ] Follow-up on action items from the **previous** retrospective has been reviewed (PM)
- [ ] At least one quality or process improvement has been identified for implementation (QA Lead or PM)
- [ ] Process documentation updated to reflect any agreed changes (Technical Writer)
- [ ] Retrospective summary shared with all team members and stakeholders (PM)

## Continuous Improvement Culture
- Measure impact of action items
- Celebrate improvements and make small, iterative changes
- Revisit the RACI and role definitions in [octoacme-roles-and-personas.md](octoacme-roles-and-personas.md) after major team changes or recurring accountability gaps
